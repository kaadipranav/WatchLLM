Execution Flow

How WatchLLM runs in three moves

Attach once, choose adversarial pressure, and get deploy-blocking evidence your infra team can actually act on.

01
Wire Decorator

Instrument real runtime

Add one decorator on the production agent path and keep your stack intact. WatchLLM captures behavior where it actually matters.

@chaos(key="sk_live_xxx")
def support_agent(request):
    return router(request)
02
Define Attack Scenarios

Select failure classes

Load targeted adversarial suites for your risk profile. Injection, memory drift, tool misuse, and jailbreak behavior run under production-like constraints.

watchllm run agent.py \
  --attacks prompt_injection,goal_hijacking,tool_abuse \
  --suite prod-critical
03
Get Failure Reports

Gate deploy on breakage

Receive trace-linked failure reports with rule IDs, risky tool calls, and severity. Stop merges until the critical path runs clean.

watchllm report latest \
  --fail-on "severity>=4" \
  --format infra
